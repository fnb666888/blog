# /post — MD 文档智能导入技能

将任意 MD 文档（书籍摘抄、AI对话记录、技术笔记等）导入博客，自动识别内容类型、优化排版、生成标题和元数据。

## 使用方式

用户调用格式：

```
/post "path/to/document.md"
/post "path/to/document.md" "自定义标题"
```

参数：

- **文档路径**（必填）：相对或绝对路径，指向用户的 MD 文件
- **自定义标题**（可选）：若不提供，根据文件名或内容自动生成

## 执行步骤

收到调用后，按以下步骤执行：

### 第 1 步：读取源文件

使用 Read 工具读取用户提供的 MD 文件。

### 第 2 步：分析内容类型并优化排版

根据文件内容特征，判断内容类型并执行对应的排版优化：

#### A. AI对话记录类（特征：问答结构或对话标记）

**识别特征**：
- 包含"问："、"答："、"Q:"、"A:"、"User:"、"Assistant:" 等对话标记
- 或者文件名包含"对话"、"chat"、"conversation"等关键词
- 或者多轮问答结构

**优化策略**：
1. 保持原有对话格式
2. 去除多余空行（超过 2 行压缩为 2 行）
3. 确保对话标记前后有适当空行
4. 如果有代码块，保持原样

**特殊处理（基于主题的 slug）**：
- 从标题中提取核心关键词生成 slug
- 中文标题转为英文翻译（如 `Hugo 部署到 Netlify` → `hugo-deploy-to-netlify`）
- 如果该 slug 的文章已存在，追加 `-2`、`-3` 等后缀创建新文章

#### B. 书籍摘抄/金句类（特征：多条独立的短句段落）

**识别特征**：
- 大量二级标题（`##`）或短句段落
- 每条独立成段，无明显逻辑关联
- 句子较短，类似格言警句

**优化策略**：
1. **将所有二级标题（`##`）转换为正文段落**（每条句子独立成段）
2. 去除文件末尾多余空行（保留 2 行以内）
3. 统一句末标点：如果原句无标点，添加句号；如果是问号/感叹号则保留
4. 每条语录之间保持单行空行分隔

#### C. 技术笔记类（特征：代码块、列表、技术术语）

**识别特征**：
- 包含代码块（```）
- 包含列表结构（- 或 1.）
- 技术术语密集（如 API、函数名、配置项）

**优化策略**：
1. 保持代码块缩进和格式
2. 统一列表符号（优先使用 `-`）
3. 为代码块添加语言标识（如果缺失）
4. 调整标题层级（确保从 `##` 开始）

#### D. 长文/思考类（特征：段落连贯，有叙事结构）

**识别特征**：
- 段落较长（超过 100 字）
- 有明显的逻辑递进
- 包含连接词（因此、所以、但是等）

**优化策略**：
1. 保持段落结构
2. 为长段落添加适当换行（每 3-5 句一段）
3. 为大段文字添加小标题（如果缺失）
4. 优化标题层级（`##` → `###` → `####`）

#### E. 混合类（无明显特征或包含多种类型）

**优化策略**：
1. 去除多余空行（超过 2 行的压缩为 2 行）
2. 统一标题格式（确保标题前后有空行）
3. 统一列表格式
4. 保持代码块原样

### 第 3 步：生成标题和描述

根据以下优先级确定标题：

1. **用户提供的自定义标题** — 直接使用
2. **文件名推断**：
   - **AI对话类** — 从对话内容中智能提取核心主题：
     - 读取文件前 500-800 个字符
     - 查找第一个问题标记（"问："、"Q:"、"User:" 等）
     - 提取问题的核心内容，去除疑问词（如何、什么、为什么等）和语气词
     - 生成简洁标题（10-25 字符）
     - 示例：
       - "问：Hugo 如何部署到 Netlify？" → "Hugo 部署到 Netlify"
       - "问：如何优化博客加载速度？" → "博客加载速度优化"
       - "User: How to setup Claude Code?" → "Claude Code 配置"
       - "问：Python 装饰器的原理是什么？" → "Python 装饰器原理"
     - 如果无法提取有效问题，则从前 3 个问题中提取共同主题关键词组合标题
   - 书籍摘抄类：去除扩展名和路径，转为标题格式（如 `人性的弱点.md` → `《人性的弱点》摘抄`）
   - 其他类型：直接使用文件名（去除扩展名）
3. **内容推断** — 从文件前 3-5 条内容中提取关键词生成标题

生成描述（用于 SEO 和摘要）：
- **AI对话类**：从前 2-3 个问题中提取核心话题，组合成描述（80-150 字符）
  - 格式：`关于{话题1}、{话题2}的对话记录，讨论了{要点摘要}`
  - 示例：`关于 Hugo 部署和配置的对话记录，讨论了 Netlify 部署流程、域名配置等问题`
- **书摘类**：格式为 `精选《{书名}》中的{数量}条经典语录，涵盖{主题1}、{主题2}等话题`
- **技术类**：格式为 `关于{技术主题}的学习笔记，包含{要点1}、{要点2}等内容`
- **思考类**：直接使用前 120 字符

### 第 4 步：推断分类和标签

根据内容特征自动推断：

**分类推断规则**：
- AI对话记录 → `["AI对话"]`
- 包含大量代码/技术术语 → `["技术"]`
- 书籍摘抄/引用类 → `["阅读"]`
- 个人思考/长文 → `["随笔"]`
- 工作方法/效率类 → `["生产力"]`
- 默认 → `["笔记"]`

**标签推断规则**：
- AI对话类自动添加 `["Claude", "AI", "日记"]`
- 书籍摘抄类自动添加 `["摘抄", "书籍"]` + 从标题提取的书名
- 技术类自动添加提取到的技术栈标签（如 `["Hugo", "Markdown"]`）
- 最多保留 5 个标签

### 第 5 步：生成 slug 和检查存在性

根据内容类型生成 URL 友好的 slug：

**AI对话类**：
- 从标题中提取核心关键词生成 slug：
  - 中文标题：转为英文翻译（如 `Hugo 部署到 Netlify` → `hugo-deploy-to-netlify`）
  - 英文标题：转小写，空格替换为 `-`，去除特殊字符
  - 示例：
    - "Hugo 部署到 Netlify" → `hugo-deploy-to-netlify`
    - "Python 装饰器原理" → `python-decorator-principles`
    - "Claude Code 配置" → `claude-code-setup`
- **检查是否已存在**：
  - 如果 `content/blog/{slug}/` 已存在，在末尾追加 `-2`、`-3` 等后缀
  - 每次导入都创建新文章，不再追加内容

**其他类型（书摘、技术、思考等）**：
- 中文标题：转为英文翻译（如 `人性的弱点` → `weaknesses-of-human-nature`）
- 英文标题：转小写，空格替换为 `-`，去除特殊字符
- 如果 slug 已存在，在末尾追加 `-2`、`-3` 等后缀（不追加内容，创建新文章）

### 第 6 步：获取当前时间

运行以下命令获取 ISO 8601 格式的当前时间（+08:00 时区）：

```bash
date +"%Y-%m-%dT%H:%M:%S+08:00"
```

### 第 7 步：创建博客文章

根据第 5 步的检查结果，执行对应操作：

#### 第 7a 步：创建新文章

创建目录 `content/blog/{slug}/`，然后创建 `index.md`，使用以下模板：

```yaml
---
title: "{标题}"
description: "{描述}"
summary: "{描述的前 150 个字符}"
date: {当前时间 ISO 格式}
lastmod: {当前时间 ISO 格式}
draft: false
weight: 50
categories: [{推断的分类}]
tags: [{推断的标签}]
contributors: []
pinned: false
homepage: false
params:
  seo:
    title: ""
    description: ""
    canonical: ""
    robots: ""
---

{优化后的正文内容}
```

#### 第 7b 步：追加内容到既有文章（仅限 AI对话类，同日期多次导入）

如果 `content/blog/ai-chat-YYYY-MM-DD/index.md` 已存在：

1. 使用 Read 工具读取现有文件
2. 在文件末尾追加新对话内容（加 `---` 分隔符和小标题）
3. 更新 front matter 中的 `lastmod` 字段为当前时间
4. 使用 Edit 工具更新文件

追加格式：

```markdown
---

## 补充对话（{当前时间}）

{新的对话内容}
```

### 第 8 步：确认结果

输出导入结果信息：

```
✅ MD 文档已导入并优化！

📁 路径：content/blog/{slug}/index.md
📝 标题：{标题}
📊 内容类型：{识别的类型}（AI对话/书摘/技术/思考/混合）
📏 优化项：{执行的优化项列表}
📅 日期：{日期}
📌 状态：已发布（draft: false）

后续步骤：
1. 本地预览确认排版：npm run dev
2. 如需调整，手动编辑文件或重新导入
3. 添加特色图片（可选）：在文章目录中放入 featured.png 或 featured.jpg
4. 运行部署检查：/deploy-check
5. 推送到 Git 主分支，Netlify 自动部署
```

## 排版优化示例

### 示例 1：书摘类

**原始内容**：
```markdown
## 想他人之所想

## 不在意结果

## 我们把次序弄错了——我们是希望别人先来喜欢我们，却不曾想到要如何才能让人喜欢。




```

**优化后**：
```markdown
想他人之所想。

不在意结果。

我们把次序弄错了——我们是希望别人先来喜欢我们，却不曾想到要如何才能让人喜欢。
```

### 示例 2：AI对话类

**原始内容**：
```markdown
## 2026-06-11

问：Hugo 如何部署到 Netlify？

答：只需在项目根目录创建 netlify.toml...

问：部署后如何查看日志？

答：在 Netlify 控制台的 Deploys 页面...
```

**优化后**（保持原有格式）：
```markdown
## 2026-06-11

问：Hugo 如何部署到 Netlify？

答：只需在项目根目录创建 netlify.toml...

问：部署后如何查看日志？

答：在 Netlify 控制台的 Deploys 页面...
```

**生成的标题**：`Hugo 部署到 Netlify`（从第一个问题提取核心主题）  
**生成的 slug**：`hugo-deploy-to-netlify`  
**生成的描述**：`关于 Hugo 部署和日志查看的对话记录，讨论了 Netlify 部署流程、日志查看方法等问题`  
**如果相同主题再次导入**：会生成 `hugo-deploy-to-netlify-2` 等后缀，创建新文章

## 注意事项

- 文章默认 `draft: false`（直接发布），如需草稿状态请告知我修改
- 排版优化会保留原始内容的核心格式，不会改变语义
- **AI对话类特殊规则**：
  - 从对话内容中智能提取核心主题作为标题
  - slug 基于标题生成（中文转英文翻译）
  - 如果相同主题再次导入，会追加数字后缀（如 `-2`）创建新文章
  - 自动归类到 `["AI对话"]` 分类和 `["Claude", "AI", "日记"]` 标签
- **其他类型**：如果同名 slug 已存在，会自动追加数字后缀创建新文章
- 文件路径可以是相对路径（相对于 `blog/` 根目录）或绝对路径
- 如果内容识别有误，可以通过自定义标题和手动调整来纠正
- 图片引用：如果源文件包含图片路径，需确保图片也复制到 `assets/images/` 或文章目录
